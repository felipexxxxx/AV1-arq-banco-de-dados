1) Alteracao da versao atual (NB ajustavel para reduzir overflow)

A alteracao foi feita em hash_index.py, funcao calculate_bucket_count (linha 38), com controle pela GUI.

Antes: NB = (NR // FR) + 1 Agora: alvo por bucket = int(FR * fator) (minimo 1, fator entre 0 e 1) NB = (NR // alvo) + 1

Exemplo pratico:

  • Se NR = 466000 e FR = 4:
  • Fator 1.0: alvo=4, NB ~= 116501
  • Fator 0.6: alvo=2, NB ~= 233001
  • Fator 0.5: alvo=2, NB ~= 233001

Isso continua respeitando a regra do PDF: NB > NR/FR.

2) Estrutura de dados usada no projeto

  • Dados (tabela): dicionario dataset com records, pages, nr, page_count.
  • Indice: dicionario index com bucket_count, bucket_capacity, buckets, metricas.
  • Buckets: lista com tamanho NB. Cada bucket tem:
    { "primary": [ {"key": palavra, "page_number": n}, ... ], "overflow": [ [itens], [itens], ... ] }
  • Busca: resultado de indice e scan sao dicionarios para a GUI mostrar custo, tempo e pagina.

5) Exemplo pratico completo

Cenario: buscar "zwitterionic".

  • 1) Usuario digita palavra e clica Buscar via Indice (gui.py:265).
  • 2) search_in_index() calcula bucket com hash (hash_index.py:182).
  • 3) Procura no primario; se nao achar, entra em buckets overflow (hash_index.py:197).
  • 4) Quando encontra, pega page_number e confirma na pagina real (hash_index.py:212-217).
  • 5) GUI mostra: status, bucket, leituras, custo, tempo, pagina e conteudo do bucket (gui.py:592).
  • 6) Se clicar Table Scan, o sistema le pagina 1,2,3... ate achar (metrics.py:16).
  • 7) Painel comparacao mostra diferenca de tempo e custo (gui.py:688).

6) Pontos importantes para explicar ao professor

  • O projeto nao usa banco externo (MySQL/Postgres). E uma simulacao em memoria de paginas + indice hash estatico.
  • A hash e deterministica (nao usa hash() padrao do Python).
  • Overflow e separado de paginas de dados: pagina de dados guarda palavras; bucket overflow guarda pares chave->pagina.
  • Custo de indice foi ajustado para seguir orientacao: custo = leitura da pagina de dados.
  • Table scan so executa quando o botao e clicado.
